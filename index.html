How these numbers are calculated — click to expand
▼

Warranty status: A project's effective end date = the latest end date across all three warranty types, giving each project the benefit of its longest-running coverage. Status rules:

  • Active — latest end date is >90 days from today
  • Expiring Soon — latest end date is within 90 days
  • Expired — latest end date has passed
  • No Warranty Data — all date fields are blank, "NONE", or "Not on File"

Coverage Gap — shown as two separate KPI cards:

Docs Not on File (38 projects, $33.7M): Projects where no warranty dates were ever entered. Coverage almost certainly exists in underlying POs or sales agreements — this is a documentation gap, not a confirmed absence of coverage. $33.7M = Σ POS Material Price where status = No Warranty Data Confirmed Expired (1 project, $0): One project — Grasshopper Solar, Job #194585 — has a recorded warranty end date of July 29, 2022, now 1,357 days lapsed. The POS Material value on record is $0, so no financial exposure can be quantified from the data. Review independently. Expired if: latest warranty_end_date < today Note: "POS Material Price" is a field name in the dataset — it represents the sold material value, not a cost or liability estimate.

Contract type classification: Determined directly from the dataset. Projects where POS Price – Labor = $0 or blank are classified as Material Only. All others are Self Performed. This distinction matters for warranty liability: material-only projects carry equipment failure risk only, while self-performed projects carry equipment + workmanship + installation liability. Material Only: POS_Labor = 0 → lower warranty obligation Self Performed: POS_Labor > 0 → broader warranty obligation

Days Remaining: Calculated as Latest Warranty End Date − April 16, 2026 in calendar days. Large values (e.g. 5,000–7,000 days) reflect long-term racking or component warranties that run 15–20 years — standard for commercial solar installations. Negative values = coverage has lapsed. "—" = no date on record.

These 38 projects are flagged because, but because no documentation exists in the system to confirm active coverage. In the absence of a recorded warranty end date, there is no way to verify whether a claim would be honored if a failure occurred today.

However, without that documentation on file, Terrasmart cannot readily defend its coverage position in a dispute, audit, or insurance review.

Understanding "Days Remaining" and the three warranty coverage types — click to expand
▼

Each project tracks up to three separate warranty types, each with its own start and end date:

  • Racking Warranty — covers the structural mounting/racking system. These are typically the longest-running, often 10–25 years from installation.
  • Foundation/Component Warranty — covers structural foundations and major mechanical components.
  • Workmanship Warranty — covers the quality of installation labor. These are typically the shortest, often 1–5 years.

Because these three clocks run independently, a project may have an expired Workmanship warranty but an active 20-year Racking warranty. The "Days Remaining" column reflects the latest end date across all three types — the longest remaining coverage the project holds.

Large days remaining values are expected: A value of 5,647 days (≈15.5 years) means that project's racking or component warranty runs until approximately 2041 — entirely normal for commercial solar. These are not errors.

Negative days remaining (shown in red) means the coverage window has already closed. −396 days means that project's warranty lapsed approximately 13 months ago.

"—" (No Warranty Data): No end dates were ever entered for these projects. The underlying warranty paperwork may exist with the manufacturer or in project files — it simply was never recorded here. These require verification before confirming exposure.
